# Pendahuluan: Membangun Web Profile Akademik dengan GitHub Pages

Selamat datang di blog saya! Posting pertama ini akan menjelaskan mengapa saya memutuskan untuk membuat web profile akademik menggunakan GitHub Pages dan bagaimana strukturnya dibangun.

## Mengapa GitHub Pages?

GitHub Pages menawarkan beberapa keuntungan sebagai platform untuk web profile akademik:

1. **Gratis** - Tidak ada biaya hosting
2. **Mudah dikelola** - Menggunakan Git untuk version control
3. **Aman** - HTTPS otomatis dengan sertifikat Let's Encrypt
4. **Fleksibel** - Mendukung Jekyll untuk static site generation
5. **Professional** - Terlihat kredibel dan modern

## Struktur Proyek

Web profile ini diorganisir dalam struktur folder yang rapi:

```
ahmadi.lham/
├── _config.yml          # Konfigurasi Jekyll
├── _layouts/            # Template HTML
│   └── default.html     # Layout utama
├── _posts/              # Artikel blog
├── assets/
│   ├── css/
│   │   └── style.css    # Custom styling
│   └── cv/              # File CV
├── index.md             # Homepage
├── publikasi.md         # Halaman publikasi
├── proyek.md            # Halaman proyek
├── blog.md              # Daftar blog
└── kontak.md            # Halaman kontak
```

## Fitur Utama

✅ **Profile Section** - Informasi pribadi dan biodata  
✅ **Publication List** - Daftar publikasi dan penelitian  
✅ **Project Portfolio** - Showcase proyek dan inisiatif  
✅ **Blog** - Tempat berbagi artikel dan refleksi  
✅ **Contact Page** - Informasi untuk menghubungi  
✅ **Responsive Design** - Bekerja baik di mobile dan desktop  
✅ **SEO Friendly** - Mudah ditemukan di search engine  

## Setup Awal

Untuk menggunakan template ini:

```bash
# Clone repository
git clone https://github.com/pacekonathyo/ahmadi.lham.git
cd ahmadi.lham

# Edit _config.yml dengan informasi Anda
# Edit index.md dan halaman lainnya
# Commit dan push
git add .
git commit -m "Customize profile"
git push origin main
```

## Customization

Anda dapat mengubah:
- Informasi pribadi di `_config.yml`
- Warna dan styling di `assets/css/style.css`
- Konten di file `.md` masing-masing
- Layout di `_layouts/default.html`
